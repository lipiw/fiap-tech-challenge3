Assistente Médico com IA

Sistema de suporte à decisão clínica para emergências hospitalares — combinando Fine-Tuning, RAG e orquestração de fluxos via LangGraph.

01 — Visão Geral

O que é o projeto

Desenvolvimento de um Assistente Médico inteligente para auxiliar médicos no atendimento de emergência, capaz de analisar sintomas, laudos laboratoriais e sugerir condutas com base nos protocolos internos do hospital.

Usuário Principal

Médico de atendimento da emergência hospitalar sem especialidade definida, que atende o paciente, coleta sintomas e solicita apoio ao assistente para conduta clínica adequada.

Papel do Sistema

O assistente atua como suporte à decisão clínica — não substitui o médico, mas responde dúvidas, sugere procedimentos e emite alertas com base em protocolos internos do hospital.

Base de Conhecimento

Protocolos internos do hospital armazenados em banco vetorial para recuperação semântica (RAG), complementados por Fine-Tuning local com dados clínicos específicos.

Orquestração Inteligente

Fluxos de decisão não-lineares via LangGraph roteiam automaticamente cada intenção do médico para o agente correto — RAG semântico ou análise GPT-4o com CTCAE.

02 — Objetivos

O que o sistema deve fazer

Funcionalidades principais que o Assistente Médico deve entregar ao final do projeto.

Consulta Clínica Contextualizada

Receber dados do paciente do médico e responder dúvidas com base em protocolos internos, mantendo contexto durante o atendimento.

Classificação de Intenções

Identificar automaticamente se o médico está relatando sintomas do paciente ou submetendo um laudo laboratorial, para rotear ao fluxo correto.

Análise de Sintomas (RAG)

Buscar semanticamente nos protocolos e dados do hospital para sugerir condutas clínicas baseadas nos sintomas informados pelo médico.

Análise de Exames CTCAE

Processar laudos de hemograma e outros exames laboratoriais via GPT-4o, classificando alterações conforme a escala CTCAE (grau 1 a 5).

Alertas à Equipe Médica

Emitir alertas automáticos para a equipe quando exames indicarem valores críticos ou quando o quadro clínico exigir atenção imediata.

Segurança & Auditoria

Registrar todas as interações com rastreabilidade completa — quem perguntou, o quê foi respondido, quais fontes foram usadas e quando.

05 — Análise de Exames

Escala CTCAE

Ao receber um laudo laboratorial, o sistema aciona o GPT-4o que classifica cada alteração encontrada conforme os Critérios de Terminologia Comum para Eventos Adversos (CTCAE).

G1
Leve
Assintomático / sintomas mínimos

Alteração laboratorial sem impacto clínico significativo. Conduta: monitoramento.

G2
Moderado
Intervenção mínima indicada

Limitação leve das atividades. Pode requerer ajuste de medicação ou repetição do exame.

G3
Grave
Intervenção hospitalar indicada

Limitação severa. Requer hospitalização ou intervenção médica urgente. Alerta emitido.

G4
Risco de Vida
Intervenção urgente necessária

Condição com risco de vida imediato. Alerta crítico emitido à equipe médica.

G5
Óbito
Relacionado ao evento adverso

Grau máximo da escala. Registro crítico em auditoria e notificação imediata.

Exemplo: Hemograma

  • Neutrófilos: 900/mm³ → G3
  • Hemoglobina: 8.5 g/dL → G2
  • Plaquetas: 25.000 → G4 — Alerta

07 — Premissas, Restrições & Fora do Escopo

Limites do projeto

Definições que garantem clareza de responsabilidades e evitam expansão indevida de escopo.

Premissas

  • Protocolos internos do hospital já existem em formato digital
  • O hospital fornecerá acesso aos dados clínicos anonimizados para Fine-Tuning
  • O médico é o responsável final por toda decisão clínica
  • Infraestrutura de servidores disponibilizada pelo hospital
  • Acesso à API do GPT-4o com contrato vigente

Restrições

  • Sistema não toma decisões clínicas autônomas — apenas sugere
  • Todas as sugestões devem citar a fonte (protocolo) de origem
  • Dados de pacientes não podem sair do ambiente hospitalar (on-premises ou VPC)
  • Conformidade obrigatória com LGPD e normas do CFM
Fora do Escopo — não será entregue neste projeto
Especialidades Médicas
O assistente não será configurado para especialidades específicas nesta fase. Foco exclusivo na emergência geral.
Prescrição Automática
O sistema não gera prescrições médicas. Sugere condutas baseadas em protocolos, mas não emite documentos clínicos.
Integração com HIS/Prontuário
Integração com sistemas de prontuário eletrônico (HIS) não está no escopo desta fase.
Atendimento ao Paciente
O assistente se comunica exclusivamente com o médico. Não há interface de atendimento direto ao paciente.
App Mobile
A interface web (desktop/tablet) é o canal de entrega desta fase. Aplicativo mobile é escopo futuro.
Análise de Imagens
Processamento de imagens de exames (raio-X, tomografia, etc.) não faz parte desta fase do projeto.
